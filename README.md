# Práctica de Git

## 1️⃣ ¿Qué es Git?
Git es un sistema de control de versiones que permite gestionar cambios en archivos y proyectos.

## 2️⃣ ¿Cuál es el flujo de trabajo de Git?
- Working Directory
- Staging Area
- Repository

Pasos:
1. Modificar archivos
2. git add
3. git commit
4. git push

## 3️⃣ Comandos realizados

- git add
- git commit
- git push
- git pull
- git fetch
- git merge

## 4️⃣ Conflictos en Git

Un conflicto ocurre cuando dos cambios afectan la misma parte de un archivo.

### Causas:
- Ediciones simultáneas
- Trabajo en ramas diferentes

### Solución:
1. Editar el archivo manualmente
2. Guardar cambios
3. git add .
4. git commit -m "Agregar README.md"
git push
